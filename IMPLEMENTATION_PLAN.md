# 魔法账单 - 协作记账功能实施方案

> 核心目标：将现有的"古灵阁事件"升级为支持多人协作记账的"共享事件"

---

## 一、核心概念

### 1.1 事件类型

| 类型 | 本地事件 | 共享事件（契约） |
|-----|---------|----------------|
| 数据存储 | 本地 | 云端 |
| 参与者 | 本地巫师 | 云端成员 |
| 账单同步 | 不支持 | 自动同步 |
| 创建者 | 单人 | 守护者 |

### 1.2 参与者类型

| 类型 | 来源 | 唯一ID | 微信授权 |
|-----|------|--------|---------|
| 微信用户 | 通过邀请码加入 | openid | ✅ 有头像昵称 |
| 自定义巫师 | 本地创建 | 本地UUID | ❌ 需设置头像 |

### 1.3 吼叫信机制（后续实现）

- 新成员申请加入 → 守护者审批
- 未审批时 → 自定义巫师可以被添加但标记为"待审批"
- 守护者审批后 → 正式成为参与者

## 二、数据模型

### 2.1 共享事件（contracts 表）

```typescript
interface SharedEvent {
  _id: string
  name: string                          // 事件名称
  guardianId: string                     // 守护者ID（创建者）
  guardianOpenid: string                // 守护者openid

  // 参与者列表
  participants: [{
    id: string                        // openid 或 自定义巫师UUID
    type: 'wechat' | 'custom'         // 微信用户 或 自定义巫师
    nickname: string                   // 昵称
    avatar: string                     // 头像URL
    role: 'guardian' | 'member'       // 守护者 或 成员
    status: 'active' | 'pending'       // active=正式成员, pending=待审批
    joinedAt: Date                     // 加入时间
    invitedBy?: string                  // 邀请人ID
  }]

  inviteCode: string                   // 6位邀请码
  status: 'active' | 'settled' | 'archived'

  totalAmount: number                   // 累计金额（分）
  billCount: number                    // 账单数量

  createdAt: Date
  updatedAt: Date
}
```

### 2.2 共享账单（bills 表）

```typescript
interface SharedBill {
  _id: string
  eventId: string                      // 关联的事件ID
  casterId: string                      // 施咒者ID
  casterOpenid: string                 // 施咒者openid

  spellType: 'simple' | 'multi'        // 均分咒 或 清算咒
  eventName: string                    // 事件名称
  totalAmount: number                  // 总金额（分）

  participants: [{
    id: string                        // 参与者ID
    name: string                      // 参与者名称
    avatar: string                    // 头像
    paid: number                      // 已付金额（分）
    consumed: number                   // 消费金额（分）
  }]

  settlements: [{                      // 结算方案
    from: string                      // 付款方
    to: string                       // 收款方
    amount: number                    // 金额（分）
  }]

  status: 'active' | 'disputed' | 'settled'

  createdAt: Date
  updatedAt: Date
}
```

### 2.3 本地数据（保持不变）

- `magic_subledgers` - 事件列表
- `magic_bills` - 本地账单
- `wizard_companions` - 休息室巫师

## 四、功能流程

### 4.1 创建共享事件（签订契约）

```
1. 古灵阁页 → "+新建事件"
2. 开启"云端同步"开关
3. 输入事件名称
4. 自动成为守护者
5. 获得邀请码
```

### 4.2 邀请成员（签订契约）

**方式A：发送邀请码**
```
1. 共享事件详情 → 分享邀请码
2. 好友输入邀请码
3. 加入共享事件
```

**方式B：面对面邀请**
```
1. 共享事件详情 → 面对面邀请
2. 显示二维码
3. 好友扫码加入
```

### 4.3 施咒记账

```
1. 顶部选择共享事件
2. 显示参与者列表（从云端获取）
3. 选择参与者（默认全选）
4. 输入金额
5. 点击清算/均分咒
6. 账单自动同步到云端
```

### 4.4 吼叫信机制（后续实现）

```
1. 有人申请加入（邀请码）
2. 守护者收到吼叫信通知
3. 守护者审批：同意/拒绝
4. 同意后 → 正式成为参与者
```

**未审批时的处理：**
- 自定义巫师可以被施咒者添加
- 但标记为"待审批"状态
- 施咒者需承担该巫师的结算金额

## 七、注意事项

1. **UI保持不变** - 现有施咒、古灵阁、冥想盆样式完全保留
2. **数据隔离** - 本地事件和共享事件数据分离
3. **唯一身份** - 参与者通过 openid 或 UUID 唯一识别
4. **守护者权限** - 只有守护者可以管理成员和删除事件
5. **最小改动** - 优先修改逻辑，不碰现有样式
